# 建置沿革

本章记录 kigurumi 社群从活动组织到资料馆建设的制度化过程。它关注的是“如何被记住”：活动如何编号，照片如何归档，访谈如何授权，公开页面和内部资料如何分层。

!!! note "写法建议"
    先写清时间线，再解释规则为什么出现。涉及个人身份和线下地址时，只写公开版本，把细节放在受限资料中。

## 示例段落

2014年前后，成员开始把活动照片、制作笔记和维护经验集中整理。2016年夏季见面会后，活动编号规则逐渐固定，后续照片、访谈和物料可以按编号回溯。2019年，第一批口述访谈完成转写，早期社群经验开始从个人记忆转入公共资料。

相关结构化事件见 [全量大事记](../generated/events.md)。
