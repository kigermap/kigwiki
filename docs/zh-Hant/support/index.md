  <div markdown>
    <p class="archive-kicker">Workflow</p>
    <h2>貢獻流程</h2>
    <p>為了讓資料多年後仍能複核，貢獻內容應盡量說明來源、時間、公開範圍與不確定之處。目前公開頁面保持精簡，新增內容會先經過授權和結構確認。</p>
  </div>
  <ol class="archive-steps">
    <li><strong>提出線索</strong><span>說明資料類型、時間範圍、公開狀態</span></li>
    <li><strong>補全來源</strong><span>附上出處、引用格式、版本差異</span></li>
    <li><strong>整理成頁</strong><span>寫入對應語言頁面或補充授權說明</span></li>
    <li><strong>校對發布</strong><span>通過 Pull Request 審核後自動部署</span></li>
  </ol>
</section>

<section class="archive-section" markdown>
  <p class="archive-kicker">Sponsorship</p>
  <h2>支持方式</h2>
  <p>目前站點優先使用免費靜態託管和公開倉庫協作。非程式碼支持同樣重要：協助聯絡資料持有人、確認授權、撰寫圖片說明、整理術語表、維護翻譯對照，都會直接提升資料館品質。</p>

  <div class="archive-note">
    <strong>版權與授權</strong>
    <p>請只提交你擁有權利、已獲授權，或可公開引用的資料。人物隱私、未公開照片、商業稿件和私人聊天記錄不應直接發布。引用材料時盡量保留來源、作者、年份和可核驗連結。</p>
  </div>
</section>
