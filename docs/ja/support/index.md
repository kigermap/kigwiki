    <p class="archive-kicker">Workflow</p>
    <h2>貢献の流れ</h2>
    <p>何年後でも資料を再確認できるよう、貢献内容には出典、時期、公開状態、不確かな点をできるだけ明記してください。現在の公開サイトは意図的に軽量なため、新しい内容は許諾と構造の確認を先に行います。</p>
  </div>
  <ol class="archive-steps">
    <li><strong>手がかりを共有</strong><span>資料の種類、時期、公開状態を説明する</span></li>
    <li><strong>出典を補完</strong><span>出典、引用形式、版の違いを添える</span></li>
    <li><strong>ページに整理</strong><span>対応する言語ページに反映するか許諾メモを補う</span></li>
    <li><strong>校閲して公開</strong><span>Pull Request の確認後に自動デプロイする</span></li>
  </ol>
</section>

<section class="archive-section" markdown>
  <p class="archive-kicker">Sponsorship</p>
  <h2>支援の方法</h2>
  <p>現在のサイトは、無料の静的ホスティングと公開リポジトリでの協作を優先しています。コード以外の支援も重要です。資料提供者への連絡、許諾確認、画像説明の作成、用語集や翻訳対照の整備は、アーカイブの品質向上に直結します。</p>

  <div class="archive-note">
    <strong>著作権と許諾</strong>
    <p>投稿できるのは、自分が権利を持つ資料、共有許諾を得た資料、または公開引用できる資料に限ってください。個人のプライバシー、未公開写真、商業原稿、私的なチャット記録は直接公開しないでください。引用時は、出典、作者、年、確認可能なリンクをできるだけ残してください。</p>
  </div>
</section>
