  <div markdown>
    <p class="archive-kicker">Workflow</p>
    <h2>Contribution Flow</h2>
    <p>To keep materials reviewable years from now, each contribution should describe its source, date, public status, and any uncertainty. The public site is intentionally lean for now, so new content should pass permission and structure checks first.</p>
  </div>
  <ol class="archive-steps">
    <li><strong>Share a Lead</strong><span>Describe the material type, date range, and public status</span></li>
    <li><strong>Complete Sources</strong><span>Add references, citation format, and version differences</span></li>
    <li><strong>Prepare the Page</strong><span>Write the matching language page or add permission notes</span></li>
    <li><strong>Review and Publish</strong><span>Deploy automatically after Pull Request review</span></li>
  </ol>
</section>

<section class="archive-section" markdown>
  <p class="archive-kicker">Sponsorship</p>
  <h2>Ways to Support</h2>
  <p>The site currently prioritizes free static hosting and public repository collaboration. Non-code support matters too: contacting material holders, confirming permission, writing image captions, maintaining glossaries, and keeping translation references all improve the archive directly.</p>

  <div class="archive-note">
    <strong>Copyright and Permission</strong>
    <p>Please submit only materials that you own, are authorized to share, or can publicly cite. Personal privacy, unpublished photos, commercial manuscripts, and private chat records should not be published directly. When citing material, preserve the source, author, year, and a verifiable link whenever possible.</p>
  </div>
</section>
